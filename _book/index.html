typescript-starter中文文档

一个用于构建javascript库和项目的typescript启动器：

  • 写标准的，未来的javascript - 具有稳定的es7功能 - 今天（stage 3或finished功能）
  • 可以选择使用typescript来改进tooling，linting和文档生成
  • 为消费者使用es6导入（如Rollup或Webpack 2），导出为一个JavaScript模块，使您的工作fully tree-shakable
  • 导出类型声明，以提高您的下游开发经验
  • Node.js风格（CommonJS）导入的向后兼容性
  • 兼备严格和灵活两种可利用的typescript配置

因此我们可以拥有很棒的东西：

  • 生成API文档（HTML或JSON），而没有一团糟的JSDoc标记来维护
  • 用AVA进行并置，原子，并发单元测试
  • 用nyc源代码覆盖率报告
  • 可配置的代码覆盖测试（用于持续集成）

开始吧

在开始之前，请考虑使用一个具有良好typescript支持的编辑器。

VS Code（以下）是一个受欢迎的选择。具有typscript支持的编辑器可以提供有用的自动完成、内嵌文档和代码重构功能。

要了解这个启动器如何用作其他项目的依赖关系，请查看示例文件夹。上面的例子来自examples/node-typescript。

开发禅宗

这个启动器包括一个监视任务，使开发更快，更具互动性。对TDD/BDD工作流程尤其有用。

要开始工作，安装Yarn并运行：

yarn watch

这将构建和监视整个项目的更改（对库源文件和测试源文件）。在开发新功能时，您可以为新功能添加测试－最初会失败－在开发新的功能之前。每次保存时，任何更改都将被重新生成并重新测试。

由于只有更改的文件被重建和重新测试，即使对于大型项目，此工作流仍然很快。

启用强类型检查（推荐）

为了让开始更容易，默认的tsconfig.json正在使用config/tsconfig.flexible配置。这将允许您开始，而没有来自Typescript的任何警告。

要启用附加的Typescript类型检查功能（对关键任务或大型项目来说是个好主意），请将tsconfig.json中的extend值更改为./config/tsconfig.strict。

查看测试覆盖率

要生成和查看测试覆盖率，请运行：

yarn cov

这将创建一个测试覆盖率的HTML报告 - 源映射回Typescript，并在默认浏览器中打开它。

生成您的API文档

使用typedoc，这个src文件夹被分析并且自动生成文档。

yarn docs

此命令以HTML格式为您的库生成API文档，并在浏览器中打开它。

由于类型由Typescript跟踪，因此不需要指定JSDoc格式的类型。有关更多信息，请参阅typedoc文档。

要生成并发布您的文档到GitHub Pages，请使用以下命令：

yarn docs:publish

一旦发布，对于您的仓库，您的文档应该在正确的GitHub页面URL中可用。 以这个仓库的GitHub页面为例。

对于更高级的文档生成，您可以提供自己的typedoc theme或构建您自己的文档使用JSON typedoc导出：

yarn docs:json
